# One-Click Self-Hosting

## Problem
Self-hosting Gratonite currently requires manually creating config files, building Docker images, debugging migration issues, and understanding federation plumbing. We want anyone — including non-technical users — to run their own instance.

## Solution
Three deployment paths, each truly one-click for its audience:

### Path 1: Local Desktop App (Mac/Windows/Linux)
Download → double-click → running. For users who want to try Gratonite on their own machine.

**How it works:**
- Tauri app (~15MB) that bundles the orchestration layer
- On first launch: checks for Docker Desktop (prompts install if missing), pulls GHCR images, generates all config, starts the stack
- System tray icon shows status (running/stopped/updating)
- Opens `http://localhost:4000` in default browser automatically
- Admin credentials shown on first-launch screen
- Federation connects via relay automatically — no config needed
- "Stop" / "Start" / "Update" from tray menu
- Data persists in `~/.gratonite/` (Docker volumes)

**Tech:** Tauri (Rust + web frontend). Lightweight, cross-platform, no Electron bloat.

### Path 2: VPS Deploy Button (gratonite.chat/deploy)
Click deploy → pick provider → enter domain → running. For users who want a public instance.

**How it works:**
- Web page at `gratonite.chat/deploy` with provider buttons (Hetzner, DigitalOcean, Vultr)
- OAuth flow or API key entry for the chosen provider
- We provision a $4-6/mo server via their API
- User enters their domain + email
- Server auto-configures: Docker install, image pull, config generation, Caddy TLS
- Redirects to `https://their-domain.com` when ready (~3 min)
- Post-deploy: shows admin credentials, basic setup wizard

**Phase 1 (simpler):** "Deploy to Hetzner" / "Deploy to DigitalOcean" buttons using their existing 1-click app / marketplace image formats — no OAuth needed, they handle provisioning.

### Path 3: CLI Installer (power users / homelabs)
`curl | bash` for users comfortable with a terminal.

```bash
curl -fsSL https://gratonite.chat/install | bash
```

**Local mode (zero prompts):**
- Detects OS, checks Docker, pulls images
- All defaults: localhost, auto-generated admin creds
- Prints URL + credentials when done

**Server mode (3 prompts):**
- Domain, email, password
- Everything else auto-generated
- Caddy auto-TLS via Let's Encrypt

## Docker Images (GHCR)
All paths pull pre-built images — no building from source.

| Image | Purpose | Size |
|-------|---------|------|
| `ghcr.io/coodayea/gratonite-setup:latest` | First-run: migrations, keypair, admin account | ~200MB |
| `ghcr.io/coodayea/gratonite-api:latest` | Node.js API server | ~250MB |
| `ghcr.io/coodayea/gratonite-web:latest` | Nginx serving SPA | ~50MB |

Built by GitHub Actions on push to main.

## Docker Compose Stack
Same compose file for all paths. Services:

| Service | Image | Purpose |
|---------|-------|---------|
| setup | gratonite-setup | Run once: migrations + admin + keypair |
| postgres | postgres:16-alpine | Database |
| redis | redis:7-alpine | Cache + pub/sub + job queues |
| api | gratonite-api | API server on :4000 |
| web | gratonite-web | SPA on :80 |
| caddy | caddy:2-alpine | Reverse proxy + auto-TLS |
| livekit | livekit/livekit-server | Voice/video (optional profile) |

## Federation & Discovery
Every self-hosted instance federates automatically:

1. **Relay connection** — On boot, API connects outbound to `wss://relay.gratonite.chat`. Ed25519 keypair (generated by setup container) authenticates the connection. No ports to open.

2. **Relay-assisted discovery** — After 48h of stable relay connection with no abuse reports, the relay signals the hub (gratonite.chat) that the instance is trustworthy. The instance then auto-pushes its public guilds to the hub's Discover page.

3. **Trust model:**
   - New instance: `trust_score: 30`, gray badge, NOT in Discover
   - 48h connected, no reports: `trust_score: 50`, appears in Discover
   - Admin-verified: `trust_score: 80+`, green badge, ranked higher
   - Reported/suspended: removed from Discover instantly

4. **Safety:**
   - Max 50 guilds per instance in Discover
   - Content sanitization on registration (strip HTML, validate URLs)
   - User reporting → auto-hide at threshold → admin review
   - Instance suspension by hub admin
   - Relay reputation engine tracks delivery success + abuse

## Installer Script Design

```bash
#!/usr/bin/env bash
set -euo pipefail

# 1. Detect OS + architecture
# 2. Check/install Docker
# 3. Prompt: local or server mode
# 4. Server mode: ask domain, email, password
# 5. Local mode: all defaults, zero prompts
# 6. Generate secrets (JWT, MFA key, DB password)
# 7. Write .env, docker-compose.yml, Caddyfile
# 8. docker compose pull && docker compose up -d
# 9. Wait for health check
# 10. Print success message with URL + credentials
```

Supports: macOS (arm64/amd64), Linux (arm64/amd64), Windows (WSL2).

## Config Generation
The installer generates three files:

**`.env`** — All secrets auto-generated:
```
INSTANCE_DOMAIN=localhost          # or user's domain
ADMIN_EMAIL=admin@localhost        # or user's email
ADMIN_PASSWORD=<generated>
DB_PASSWORD=<generated>
JWT_SECRET=<generated>
JWT_REFRESH_SECRET=<generated>
MFA_ENCRYPTION_KEY=<generated>
FEDERATION_ENABLED=true
RELAY_ENABLED=true
RELAY_URL=wss://relay.gratonite.chat
```

**`docker-compose.yml`** — Standard stack (identical for all modes).

**`Caddyfile`** — Auto-TLS for server mode, internal TLS for local mode.

## Build Order
1. **GitHub Actions + GHCR images** — foundation for everything else
2. **CLI installer script** — covers both local and server, immediate value
3. **Documentation** — all paths documented
4. **Relay-assisted discovery** — code changes to relay + hub API
5. **VPS deploy button** — Hetzner/DO marketplace or deploy page
6. **Desktop app** — Tauri app (biggest effort, highest wow-factor)

## Success Criteria
- Local: `curl install | bash` → instance running in <3 minutes, zero prompts
- Server: `curl install | bash` → instance running in <5 minutes, 3 prompts
- Federation: instance appears in Discover within 48h automatically
- Desktop: download app → double-click → running in <3 minutes
